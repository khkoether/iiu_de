Abstract

Die folgenden Schritte stellen die Funktionsfähigkeit der Installation von JRuby und Ruby on Rails (kurz: Rails) sicher.

Der Artikel ist in AsciiDoc geschrieben und mit Asciidoctor 1.5.2 erstellt worden.

Zu beachten ist, dass so ein Dokument zum Einen nie abschliessend sein kann, und zum Anderen bestimmte Konfigurationen anders, oder auf einem anderem Weg durchgeführt werden können. Gerne nehme ich Verbesserungsvorschläge entgegen.

Es gilt wie immer: Verwendung der Anleitung auf eigene Gefahr.

Vorraussetzungen

JRuby und Ruby on Rails sind wie in

  • IiU — JRuby 1.7.18, rails-4.2.0

beschrieben durchgeführt worden.

JRuby 1.7.18
$ which jruby

$ jruby -v
jruby 1.7.18 (1.9.3p551) 2014-12-22 625381c on OpenJDK 64-Bit Server VM 1.7.0_65-b32 +jit [linux-amd64]

$ jruby -S rails -v   
Rails 4.2.0

$ rails -v     
Rails 4.2.0
Der Schalter -S führt dazu, daß für das Script zuerst das Verzeichnis ${JRUBY_HOME}/bin geprüft wird - und dann der PATH.
Solange wie die ausführbaren Programme von JRuby on Rails im PATH vorangestellt werden, kann auf jruby -S verzichtet werden.

Datenbanksystem wählen

Für das Framework Ruby on Rails sind die folgenden drei Datenbanksysteme zumindest für die Entwicklung (aber auch für die Produktion) eines neuen Rails-Projektes die erste Wahl:

  • SqLite3
    Klein, kompakt und dateibasiert – mit diesen Vorzügen glänzt SQLite3.
    Ist in Ruby on Rails die Default-Datenbank.

  • MySQL
    Gut und schnell und überhaupt …​ überall verfügbar.

  • PostgreSQL
    Lupenreine OpenSource-Datenbanksystem für professionelle Anforderungen.

Die einzelnen Schritte für die Datenbank PostgreSQL werden in diesem Artikel nicht beschrieben.
In der Vorgehensweise sind sie ähnlich derer für MySQL

Datenbankadapter konfigurieren

config/database.yml: sqlite3
[...]
development:
  adapter: sqlite3
  database: db/development.sqlite3   
[...]
Die Datenbank-Datei wird mit dem ersten Aufruf automatisch angelegt.
Keine Angabe für Benutzer und Passwort!
Datei: config/database.yml mysql
[...]
default: &default
  adapter: mysql
  encoding: utf8
  pool: 5
  username: sid                 <---
  password: g3h3im              <---
  socket: /var/run/mysqld/mysqld.sock

development:   
  <<: *default
  database: jmyapp_development   
[...]
Es werden ausschließlich die zu default geänderten Einträge aufgeführt.
Der hier vorgeschlagene Datenbank-Name wird von Ruby on Rails automatisch gebildet:
Name der Applikation MyApp und dem Zusatz _development
